# DeepFilterNet：频域深度滤波

官方仓库：[Rikorose/DeepFilterNet](https://github.com/Rikorose/DeepFilterNet)

## 概述

DeepFilterNet 是 2022 年德国 Fraunhofer IIS 的 Hendrik Schröter 等人提出的语音增强方法。在 DNS Challenge 2022 上取得了优异成绩，同时计算量控制得当，是性能和效率之间的良好平衡。

核心创新是 **深度滤波 (Deep Filtering)**：不是简单地给每个频点乘一个增益（传统掩码方法），而是预测一个时变的 FIR 滤波器，对相邻几帧做卷积。这样能利用时间上下文，同时隐式地处理相位。

---

## 一、和 RNNoise 的区别

| | RNNoise | DeepFilterNet |
|---|---------|---------------|
| 核心思路 | 频带增益 | 深度滤波 |
| 相位处理 | 不处理 | 隐式处理 |
| 网络 | 纯 GRU | 编码器 + GRU + 双路径解码器 |
| 频率处理 | 32 个 ERB 频带 | 双路径（ERB + 线性） |
| 参数量 | ~100K | ~720K (DeepFilterNet2) |
| PESQ | 2.54 | 2.89 |

DeepFilterNet 用更多参数换来更好效果，但比 DCCRN 那些动辄几百万参数的方法轻量很多。

## 三、ERB 频带

DeepFilterNet 也用了 ERB 频带分组，思路和 RNNoise 一样。

ERB（Equivalent Rectangular Bandwidth）是基于人耳听觉特性的频带划分。人耳在低频处分辨率高，高频处分辨率低，ERB 按这个特性划分。

带宽公式：
$$B_{ERB}(f_c) = 24.7 \times (4.37 \times 10^{-3} f_c + 1)$$

具体数值：
- 100Hz 处，带宽约 35Hz
- 1000Hz 处，带宽约 132Hz
- 8000Hz 处，带宽约 894Hz

为什么用 ERB 不用 Mel？在语音增强任务上，ERB 效果略好一点，而且公式更简洁。

## 四、深度滤波 (Deep Filtering)

这是 DeepFilterNet 最核心的创新。

### 传统掩码 vs 深度滤波

**传统掩码方法**：给每个时频点乘一个系数
$$\hat{S}(k, t) = M(k, t) \cdot X(k, t)$$

**深度滤波**：对相邻几帧做卷积
$$\hat{S}(k, t) = \sum_{\tau=0}^{L-1} H(k, t, \tau) \cdot X(k, t-\tau)$$

$H$ 是网络预测的滤波器系数，$L$ 是滤波器长度（DeepFilterNet 用 5）。

直观理解：
```
传统掩码：当前帧 × 系数 = 输出

深度滤波：
  当前帧×H[0] + 前1帧×H[1] + 前2帧×H[2] + 前3帧×H[3] + 前4帧×H[4] = 输出
```

### 为什么能处理相位？

传统掩码只能改变幅度，相位没法动。深度滤波用的是**复数乘法**，$H$ 和 $X$ 都是复数，相乘时相位会叠加。

而且因为用了多帧信息，网络可以学到：语音在相邻帧之间高度相关，噪声相关性低。通过合适的滤波器系数，可以增强语音、抑制噪声，同时隐式修正相位。

### DfOp 操作详解

深度滤波操作的核心模块，将 DF Decoder 输出的滤波器系数应用到频谱上。

#### 结构图

```
输入:
  - spec:  [B, 1, T, 481, 2]  完整频谱（实部+虚部）
  - coefs: [B, T, 5, 96, 2]   滤波器系数（5阶，96频点，复数）
  - alpha: [B, T, 1]          混合系数
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│  步骤1: 提取低频部分                                      │
│  spec_f = spec[..., :96, :]  → [B, 1, T, 96, 2]        │
└─────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│  步骤2: 时间轴 padding（获取历史帧）                      │
│  pad(spec_f, left=4, right=0)  → [B, 1, T+4, 96, 2]    │
│                                                          │
│  示例（df_order=5, lookahead=0）:                        │
│  原始: [t0, t1, t2, ..., tN]                            │
│  填充: [0, 0, 0, 0, t0, t1, t2, ..., tN]                │
└─────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│  步骤3: Unfold 展开滑动窗口                               │
│  unfold(dim=T, size=5, step=1)                          │
│  → [B, 1, T, 96, 2, 5]                                  │
│                                                          │
│  每个时刻 t 现在包含 [t-4, t-3, t-2, t-1, t] 五帧       │
└─────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│  步骤4: 复数乘法（核心滤波操作）                          │
│                                                          │
│  padded: [B, 1, T, 96, 2, 5]                            │
│  coefs:  [B, T, 5, 96, 2] → permute → [B, 2, T, 96, 5] │
│                                                          │
│  复数乘法: (a+bi)(c+di) = (ac-bd) + (ad+bc)i            │
│                                                          │
│  out_re = padded[...,0,:] × coefs[:,0:1]                │
│         - padded[...,1,:] × coefs[:,1:2]                │
│                                                          │
│  out_im = padded[...,0,:] × coefs[:,1:2]                │
│         + padded[...,1,:] × coefs[:,0:1]                │
│                                                          │
│  → [B, 1, T, 96, 5] (实部和虚部)                         │
└─────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│  步骤5: 对滤波器阶数求和                                  │
│  out_re = sum(out_re, dim=-1)  → [B, 1, T, 96]         │
│  out_im = sum(out_im, dim=-1)  → [B, 1, T, 96]         │
│                                                          │
│  stack → spec_f [B, 1, T, 96, 2]                        │
└─────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│  步骤6: Alpha 混合                                        │
│                                                          │
│  spec_out = spec_f × alpha + spec[..., :96, :] × (1-α) │
│                                                          │
│  - alpha 接近 1: 使用滤波后的结果                        │
│  - alpha 接近 0: 保留原始频谱                            │
└─────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│  步骤7: 替换回完整频谱                                    │
│  spec[..., :96, :] = spec_out                           │
│  spec[..., 96:, :] 保持不变（高频不处理）                │
└─────────────────────────────────────────────────────────┘
    │
    ▼
输出: spec [B, 1, T, 481, 2]  增强后的频谱
```

#### 关键理解

**1. 为什么用 Unfold？**
- 把时间轴展开成滑动窗口，每个时刻包含当前帧和前 4 帧
- 这样可以用向量化操作代替循环，提高效率

**2. 复数乘法的作用**
- 不仅改变幅度，还会影响相位
- 通过学习合适的滤波器系数，网络可以同时优化幅度和相位

**3. Alpha 的作用**
- 控制滤波强度，避免过度处理
- 网络可以学会在不同情况下使用不同的混合比例
- 高 SNR 时 alpha 小（轻度处理），低 SNR 时 alpha 大（强力处理）

**4. 为什么只处理低频 96 个频点？**
- 低频（0-5kHz）是语音的主体，需要精细处理
- 高频主要是噪声和辅音，ERB 掩码的粗粒度处理就够了
- 节省计算量

**5. 迭代应用（df_n_iter=2）**
```
spec_0 = 原始频谱
spec_1 = DfOp(spec_0, coefs, alpha)  # 第一次滤波
spec_2 = DfOp(spec_1, coefs, alpha)  # 第二次滤波（用同样的系数）
```
- 迭代可以逐步细化结果
- 类似于多次"抛光"

### 5.2 为什么这么设计？

**双路径分工：**

| 路径 | 处理范围 | 分辨率 | 输出 | 作用 |
|------|---------|--------|------|------|
| ERB | 全频段 0-24kHz | 粗 (32 频带) | 实数增益 | 整体能量调整 |
| DF | 低频 0-5kHz | 细 (96 频点) | 复数滤波器 | 精细处理+相位 |

低频是语音的主体，需要精细处理，相位在低频更重要。高频主要是辅音和噪声，粗粒度处理就够了。

### 6.3 Encoder 设计理念深度解读

#### 为什么要双路径设计？

**1. 频率分辨率的权衡困境**

语音增强面临一个根本矛盾：
- 低频（0-2kHz）包含语音的基频和主要共振峰，需要精细处理，相位信息对语音质量影响巨大
- 高频（5-24kHz）主要是辅音和噪声，粗粒度处理就够了，过度精细反而浪费计算

如果用单一分辨率：
- 全用高分辨率（481 个频点）→ 计算量爆炸，参数量太大
- 全用低分辨率（32 个 ERB）→ 低频细节丢失，语音质量下降

双路径的解决方案：
- ERB 路径：32 个频带覆盖全频段（0-24kHz），负责整体能量分布
- DF 路径：96 个频点只处理低频（0-5kHz），负责精细相位处理

这样既保证了低频的精细度，又控制了计算量。

**2. 任务分工的必然性**

语音增强实际上是两个子任务：
- 能量调整：哪些频带该保留、哪些该抑制（粗粒度决策）
- 相位修正：如何让语音听起来自然、不失真（精细操作）

ERB 路径做"战略决策"：这个频带是语音还是噪声？
DF 路径做"战术执行"：具体怎么滤波才能既去噪又保真？

如果混在一起，网络很难学到这种层次化的表示。

**3. 信息互补性**

- ERB 特征：输入是幅度谱（实数），维度 [B, 1, T, 32]，只有能量信息
- DF 特征：输入是复数谱（实部+虚部），维度 [B, 2, T, 96]，包含相位信息

两条路径看到的是不同视角的信息：
- ERB 看到的是"整体能量分布"
- DF 看到的是"低频的精细结构"

最后通过融合让网络同时理解"宏观"和"微观"。

#### 为什么频率轴要下采样，时间轴不动？

**频率轴下采样的原因**

ERB 路径：32 → 16 → 8 → 8（压缩 4 倍）
DF 路径：96 → 48（压缩 2 倍）

- 感受野扩大：每次下采样 2 倍，感受野就扩大 2 倍。最终每个频率点能"看到"更宽的频带范围
- 跨频带建模：语音的共振峰通常跨越多个频带，下采样让网络能捕捉这种"频带间的关系"
- 参数效率：如果不下采样，flatten 后的维度会翻倍，GRU 的参数量爆炸

DF 路径只压缩 2 倍是因为它负责精细处理，不能压缩太多，否则丢失细节。96→48 是在"保留细节"和"控制参数"之间的平衡。

**时间轴不下采样的原因**

所有卷积的时间步长都是 1，为什么？

- 时序连续性：语音是高度时序相关的信号，相邻帧之间的变化很平滑。如果时间轴下采样，会丢失细粒度的时序信息
- 因果性要求：实时系统需要逐帧处理，如果时间轴下采样，会引入额外的延迟
- GRU 的作用：时序建模交给 GRU 来做，卷积只负责提取局部特征。GRU 天然就能处理长时序依赖

对比图像任务：图像的时间和空间都下采样，但语音的时间维度比图像的空间维度更"脆弱"，不能随意压缩。

#### 为什么要保留多层 skip connection（e0, e1, e2, e3）？

Encoder 输出了 4 层特征 e0~e3，全部传给 Decoder。

**多尺度信息的必要性**

| 层级 | 频率分辨率 | 包含的信息 | Decoder 用途 |
|------|-----------|-----------|-------------|
| e0 | 32 ERB | 原始频谱细节 | 恢复精细边界 |
| e1 | 16 ERB | 局部模式 | 恢复局部结构 |
| e2 | 8 ERB | 中层语义 | 恢复共振峰 |
| e3 | 8 ERB | 高层语义 | 语音/噪声判别 |

类比画画：e3 是"构图"，e2 是"轮廓"，e1 是"细节"，e0 是"笔触"。Decoder 需要所有层次才能重建完整的掩码。

**梯度流动**

如果只传 e3，Decoder 的梯度要经过很多层才能回传到 Encoder 的浅层，容易梯度消失。有了 skip connection，e0 的梯度可以直接从 Decoder 的输出层回传，训练更稳定。

**信息瓶颈问题**

e3 的维度是 [B, 16, T, 8]，flatten 后只有 128 维。如果只传 e3，相当于把 32 个 ERB 的信息压缩到 128 维，信息损失很大。有了 e0~e2，Decoder 可以"绕过"这个瓶颈，直接访问浅层的高分辨率信息。

#### 为什么两条路径最后要合并成统一的 embedding？

ERB 路径和 DF 路径最后都 flatten 成 128 维，然后相加（或拼接）成统一的 emb，再过 GRU。

**信息融合的必要性**

- ERB 路径知道"哪些频带有语音"
- DF 路径知道"低频的相位结构"

这两个信息必须融合，才能做出正确的决策。比如：
- 如果 ERB 路径判断某个频带是噪声，DF 路径就不应该对它做精细滤波
- 如果 DF 路径发现低频有强周期性（语音特征），ERB 路径应该保留对应频带

统一的 embedding 让两条路径的信息可以"对话"。

**GRU 的时序建模**

GRU 需要一个统一的输入来建模时序依赖。如果两条路径独立，就需要两个 GRU，参数量翻倍。而且，语音的时序模式是跨频段的（比如共振峰的移动），需要在融合后的表示上建模。

**Decoder 的输入一致性**

两个 Decoder（ERB Decoder 和 DF Decoder）都需要从同一个 embedding 出发，这样才能保证：
- ERB 掩码和 DF 滤波器是"协调"的
- 不会出现 ERB 掩码把某个频带置零，DF 滤波器还在对它做精细处理的矛盾

#### 为什么要预测 LSNR（本地信噪比）？

Encoder 最后输出一个 lsnr [B, T, 1]，范围 [-15, 35] dB。

**辅助训练信号**

LSNR 是一个中间监督信号。训练时可以用真实的 SNR 作为标签，让网络学会"估计当前帧有多干净"。这个任务比直接预测掩码简单，能帮助 embedding 学到更好的表示。

**自适应处理**

LSNR 可以用来动态调整处理强度：
- 高 SNR（干净）：轻度处理，避免过度抑制
- 低 SNR（很吵）：强力处理，激进去噪

**可解释性**

LSNR 让网络的决策过程更透明。如果输出质量不好，可以检查 LSNR 的预测是否准确，帮助调试。

#### 为什么 DF 路径要保留 c0 直接传给 Decoder？

注意到 DF Decoder 的输入不只是 emb，还有 encoder 的 c0 [B, 16, T, 96]。

**高频信息的保留**

emb 经过 GRU 后，维度是 [B, T, 256]，已经是高度抽象的表示。如果只用 emb，DF Decoder 很难恢复出 96 个频点的精细滤波器系数。c0 保留了原始的 96 个频点的信息，相当于给 Decoder 一个"参考答案"。

**残差学习**

DF Decoder 的输出是残差结构，网络只需要学习"在 c0 的基础上做什么调整"，而不是从头预测整个滤波器，降低了学习难度。

**梯度直通**

c0 的存在让梯度可以直接从 DF Decoder 回传到 Encoder 的 DF 路径，加速收敛。

#### 设计哲学总结

DeepFilterNet 的 Encoder 设计体现了几个核心思想：

1. **分而治之**：用双路径分别处理粗粒度和精细粒度，避免"一刀切"
2. **信息分层**：保留多层特征（e0~e3），让 Decoder 能访问不同抽象层次的信息
3. **时频非对称**：频率轴可以压缩（因为有 ERB），时间轴不能动（因为需要因果性）
4. **融合与独立的平衡**：两条路径最后融合（emb），但也保留独立信息（c0）传给 Decoder
5. **辅助监督**：LSNR 预测帮助学习更好的表示

这种设计不是"拍脑袋"想出来的，而是在计算效率、处理精度、训练稳定性之间反复权衡的结果。

相比之下：
- RNNoise 只有单路径，没有精细处理能力
- DCCRN 用 U-Net 对所有频点一视同仁，计算量大
- DeepFilterNet 找到了一个"甜蜜点"：用双路径在关键区域（低频）投入资源，其他地方节省开销

这就是为什么 DeepFilterNet 能在 720K 参数下达到接近百万参数模型的效果。

### 8.3 DF Decoder 设计理念深度解读

#### 为什么要用 3 层 GRU？

**ERB Decoder 只用 1 层 GRU，DF Decoder 却用 3 层，为什么？**

**1. 任务复杂度不同**

ERB Decoder 的任务（简单）：
- 输出：32 个实数掩码值 [0, 1]
- 含义：每个频带保留多少能量
- 这是一个分类问题：语音 vs 噪声

DF Decoder 的任务（复杂）：
- 输出：96×5×2 = 960 个复数滤波器系数
- 含义：如何对 5 帧历史做加权组合
- 这是一个生成问题：需要预测精细的滤波器参数

**2. 时序依赖的深度**

深度滤波器的系数需要考虑：
- 当前帧的语音/噪声特性
- 前几帧的变化趋势（语音在上升还是下降？）
- 跨帧的相关性（如何利用历史信息？）

3 层 GRU 的作用：
- 第 1 层：提取局部时序模式（2-3 帧的短期变化）
- 第 2 层：整合中期模式（5-10 帧的趋势）
- 第 3 层：捕获长期依赖（更长时间的上下文）

类比：
- ERB 掩码像"开关"：开或关，1 层 GRU 够了
- DF 滤波器像"调音台"：需要精细调节多个旋钮，需要更深的网络

#### 为什么输出用 Tanh 激活？

**为什么不用 ReLU 或 Sigmoid？**

**1. 滤波器系数需要正负值**

物理意义：
- 正系数：增强某个频点
- 负系数：抑制某个频点
- 零系数：不改变

如果用 ReLU（只有正值）或 Sigmoid（只有 [0,1]），就无法表达"抑制"的含义。

**2. Tanh 的范围 [-1, 1] 很合适**

为什么不用无界的激活（如不加激活）？
- 滤波器系数太大会导致数值不稳定
- Tanh 限制在 [-1, 1]，提供了天然的正则化
- 训练更稳定，不容易爆炸

**3. Tanh 的梯度特性**

- 在 0 附近梯度大，远离 0 梯度小
- 鼓励网络学习"适度"的滤波器系数
- 避免过度激进的处理

#### 为什么要有 c0 的 skip connection？

**这是 DF Decoder 最关键的设计！**

**1. 信息瓶颈问题**

问题：emb 只有 256 维，如何恢复 960 维的滤波器系数？

这是一个信息扩张的过程，从 256 维到 960 维，信息从哪来？

如果只用 emb：
- 网络必须从 256 维的抽象表示中"猜"出 960 维的细节
- 很多低频的精细信息在 GRU 处理过程中丢失了
- 预测的滤波器可能不准确

有了 c0：
- c0 [B, 16, T, 96] 保留了原始的 96 个频点信息
- 经过 1×1 卷积变成 [B, 10, T, 96]，正好对应 5×2=10（5 阶滤波器 × 复数）
- 网络只需要学习"在 c0 的基础上做什么调整"

**2. 残差学习的优势**

残差结构：coefs = Linear(emb) + Conv(c0)

为什么残差学习更容易？

不用残差（从零学习）：
- 网络需要学习：如何从头预测整个滤波器
- 学习目标：coefs = f(emb)
- 难度：高

用残差（学习调整）：
- 网络只需要学习：在 c0 的基础上做什么微调
- 学习目标：coefs = c0 + Δ(emb)
- 难度：低

类比：
- 不用残差：让你画一幅画
- 用残差：给你一张草图，让你修改

**3. 频率对齐**

c0 的维度是 [B, 16, T, 96]：
- 16 是通道数
- 96 是频点数，正好对应 DF 处理的 96 个低频频点

这不是巧合！c0 是 Encoder DF 路径的第一层输出，它保留了：
- 每个频点的原始特征
- 频点之间的相对关系
- 低频的精细结构

通过 1×1 卷积，c0 被转换成滤波器系数的"初始猜测"，然后 emb 提供的是"全局调整"。

#### 为什么要分成两个输出头？

**为什么不直接输出 961 维（960 个系数 + 1 个 alpha）？**

**1. 任务性质不同**

coefs（滤波器系数）：
- 复杂的多维输出（960 维）
- 需要精细的频点级控制
- 需要 c0 的辅助

alpha（混合系数）：
- 简单的标量输出（1 维）
- 全局的强度控制
- 不需要频点级信息

**2. 学习难度不同**

如果混在一起：
- 960 维的 coefs 会主导梯度
- 1 维的 alpha 的梯度会被"淹没"
- alpha 很难学好

分开输出：
- 各自有独立的损失和梯度
- alpha 可以专注学习"该用多强的滤波"
- coefs 可以专注学习"具体怎么滤波"

**3. 语义清晰**

alpha 的含义：
- 这一帧需要多强的处理？
- 是轻度去噪（alpha=0.3）还是强力去噪（alpha=0.9）？
- 这是一个全局决策

coefs 的含义：
- 每个频点具体怎么滤波？
- 这是一个局部决策

分开输出让网络的决策过程更清晰。

#### 为什么 alpha 用 Sigmoid？

**1. 混合系数的物理意义**

output = filtered × alpha + original × (1 - alpha)

- alpha = 0：完全保留原始频谱
- alpha = 1：完全使用滤波后的频谱
- alpha ∈ (0, 1)：两者混合

Sigmoid 的输出范围正好是 [0, 1]，完美匹配这个需求。

**2. 自适应处理强度**

网络可以学会：
- 高 SNR（干净）：alpha 小，轻度处理
- 低 SNR（很吵）：alpha 大，强力处理
- 不确定时：alpha 中等，保守处理

**3. 防止过度处理**

如果没有 alpha，滤波器可能会过度激进，导致：
- 语音失真
- 引入人工痕迹
- 听起来"不自然"

alpha 提供了一个"安全阀"，让网络可以在不确定时选择保守策略。

#### 整体设计哲学

DF Decoder 的设计体现了几个核心思想：

**1. 分层决策**

- 高层（GRU）：理解语音/噪声的整体模式
- 中层（alpha）：决定处理强度
- 低层（coefs + c0）：执行精细滤波

**2. 信息保留与抽象的平衡**

- emb：高度抽象，包含全局语义
- c0：低层细节，包含频点信息
- 两者结合：既有全局理解，又有局部细节

**3. 任务分解**

不是一次性预测 961 个数，而是：
- 先预测"要不要滤波"（alpha）
- 再预测"怎么滤波"（coefs）
- 降低学习难度

**4. 鲁棒性设计**

- Tanh 限制系数范围，防止数值爆炸
- alpha 提供混合，防止过度处理
- 残差学习降低学习难度

#### 与 ERB Decoder 的对比

| 特性 | ERB Decoder | DF Decoder |
|------|------------|-----------|
| GRU 层数 | 1 层 | 3 层 |
| 任务类型 | 重建（掩码） | 生成（滤波器） |
| 输出维度 | 32 | 960 + 1 |
| Skip connection | 4 层（e0~e3） | 1 层（c0） |
| 输出激活 | Sigmoid | Tanh + Sigmoid |
| 设计理念 | U-Net 镜像 | 残差生成 |

DF Decoder 的设计更复杂，因为它的任务更难。它需要生成精细的滤波器参数，而不只是简单的掩码值。

### 8.4 输出维度解析

```
滤波器系数 coefs: [B, T, df_order, nb_df, 2]
                  [B, T,    5,      96,   2]

- B: batch size
- T: 时间帧数
- df_order=5: 滤波器长度 (看当前帧 + 前 4 帧)
- nb_df=96: 处理的频点数 (0-5kHz)
- 2: 复数的实部和虚部

alpha: [B, T, 1]
- 混合系数，控制 "原始频谱 vs 滤波后频谱" 的权重
- 范围 [0, 1]，Sigmoid 输出
```

## 九、主网络 DfNet

DfNet 把所有模块组装起来。

### 9.1 完整流程

```python
class DfNet(nn.Module):
    def __init__(self, erb_fb, erb_inv_fb, run_df=True):
        super().__init__()
        p = ModelParams()

        self.enc = Encoder(p)
        self.erb_dec = ErbDecoder(p)
        self.df_dec = DfDecoder(p)

        # ERB → 线性频率的掩码应用
        self.mask = Mask(erb_fb, erb_inv_fb, post_filter=p.mask_pf)

        # 深度滤波操作
        self.df_op = DfOp(
            p.nb_df,
            p.df_order,
            p.df_lookahead
        )

        self.df_n_iter = p.df_n_iter  # 迭代次数，默认 2
        self.run_df = run_df

    def forward(self, spec, feat_erb, feat_spec):
        """
        spec:      [B, 1, T, F, 2]   原始频谱 (F=481)
        feat_erb:  [B, 1, T, 32]     ERB 特征
        feat_spec: [B, 2, T, 96]     DF 特征
        """
        # ===== 1. Encoder =====
        e0, e1, e2, e3, emb, c0, lsnr = self.enc(feat_erb, feat_spec)

        # ===== 2. ERB Decoder → 生成掩码 =====
        m = self.erb_dec(emb, e3, e2, e1, e0)  # [B, 1, T, 32]

        # ===== 3. 应用 ERB 掩码 =====
        # 把 32 个 ERB 增益插值到 481 个线性频点，然后乘到频谱上
        spec = self.mask(spec, m)

        # ===== 4. DF Decoder → 生成滤波器系数 =====
        df_coefs, df_alpha = self.df_dec(emb, c0)
        # df_coefs: [B, T, 5, 96, 2]
        # df_alpha: [B, T, 1]

        # ===== 5. 迭代应用深度滤波 =====
        if self.run_df:
            for _ in range(self.df_n_iter):
                spec = self.df_op(spec, df_coefs, df_alpha)

        return spec, m, lsnr, df_alpha
```

## 十一、损失函数

DeepFilterNet 用了多个损失的组合：

$$\mathcal{L} = \lambda_1 \mathcal{L}_{spec} + \lambda_2 \mathcal{L}_{multi} + \lambda_3 \mathcal{L}_{alpha}$$

### 11.1 频谱损失（Spectral Loss）

**设计理念**：同时优化幅度和相位

传统方法只优化幅度谱，但 DeepFilterNet 需要处理相位，所以损失函数包含三个部分：

**1. 幅度损失**
- 计算预测频谱和真实频谱的幅度差异
- 使用 L1 范数（而不是 L2）
- 原因：L1 对异常值不敏感，训练更稳定

**2. 实部损失**
- 直接比较复数频谱的实部
- 隐式地优化相位信息

**3. 虚部损失**
- 直接比较复数频谱的虚部
- 与实部一起完整地约束相位

**为什么这样设计？**

如果只优化幅度，相位可能会乱掉。通过同时约束实部和虚部，网络被迫学习正确的相位结构。

**权重分配**：
- 幅度损失权重较大（因为人耳对幅度更敏感）
- 实部+虚部损失权重较小（作为辅助约束）

### 11.2 多分辨率 STFT 损失（Multi-Resolution Loss）

**设计理念**：捕获不同时频分辨率的信息

使用多个不同的 FFT 大小（如 512、1024、2048）分别计算损失，然后求和。

**为什么需要多分辨率？**

**1. 时频分辨率的权衡**
- 小 FFT（512）：时间分辨率高，频率分辨率低 → 捕获快速变化
- 大 FFT（2048）：时间分辨率低，频率分辨率高 → 捕获精细频率结构

**2. 不同尺度的特征**
- 短时窗：捕获辅音、爆破音等瞬态特征
- 长时窗：捕获元音、基频等稳态特征

**3. 防止过拟合**
- 单一分辨率容易让网络"钻空子"
- 多分辨率强制网络在所有尺度上都表现良好

**具体做法**：
对每个 FFT 大小，都计算一次频谱损失（幅度+实部+虚部），然后加权求和。

### 11.3 Alpha 损失（Alpha Loss）

**设计理念**：对低能量区域更敏感

这是一个特殊的损失，使用幂次变换：

$$\mathcal{L}_{alpha} = ||\hat{S}|^\alpha - |S|^\alpha||_1$$

其中 $\alpha$ 通常取 0.3 左右。

**为什么要用幂次变换？**

**1. 压缩动态范围**

语音信号的动态范围很大：
- 高能量区域（如元音）：幅度可能是 1000
- 低能量区域（如辅音）：幅度可能是 10

如果直接用 L1 或 L2：
- 高能量区域的误差会主导损失
- 低能量区域的误差被忽略

**幂次变换的效果**：
- 1000^0.3 ≈ 8
- 10^0.3 ≈ 2

动态范围从 100:1 压缩到 4:1，低能量区域的重要性提升。

**2. 符合人耳感知**

人耳对声音强度的感知是对数的（分贝刻度），幂次变换接近这种特性。

**3. 保护弱信号**

辅音、摩擦音等弱信号对语音清晰度很重要，但能量低。Alpha 损失确保这些细节不被忽略。

### 11.4 为什么不用 MSE？

**MSE（均方误差）的问题**：

**1. 对高能量区域过度关注**
- MSE 对大误差的惩罚是平方级的
- 高能量区域的误差会"淹没"低能量区域

**2. 对相位不敏感**
- 两个幅度相同但相位不同的信号，MSE 可能很小
- 但听起来完全不同

**3. 容易产生过平滑的结果**
- MSE 倾向于预测"平均值"
- 导致输出模糊，缺乏细节

**L1 的优势**：
- 对异常值更鲁棒
- 鼓励稀疏性（有助于去噪）
- 训练更稳定

### 11.5 损失权重的平衡

典型的权重配置：

| 损失类型 | 权重 | 作用 |
|---------|------|------|
| 单分辨率频谱损失 | 1.0 | 基础约束 |
| 多分辨率损失 | 0.5-1.0 | 多尺度约束 |
| Alpha 损失 | 0.1-0.3 | 保护弱信号 |

**权重调整的考虑**：

1. **任务优先级**：如果更关心整体质量，提高频谱损失权重；如果更关心细节，提高 alpha 损失权重

2. **训练阶段**：
   - 早期：频谱损失权重大，快速收敛
   - 后期：alpha 损失权重增加，细化细节

3. **数据特性**：
   - 高 SNR 数据：可以降低 alpha 损失权重
   - 低 SNR 数据：需要提高 alpha 损失权重保护弱信号

### 11.6 辅助损失：LSNR 损失

除了主要的频谱损失，还有一个辅助损失用于训练 LSNR 预测头：

$$\mathcal{L}_{LSNR} = ||LSNR_{pred} - LSNR_{true}||_1$$

**作用**：
- 帮助 embedding 学到更好的表示
- 提供中间监督信号
- 让网络学会"估计噪声水平"

**权重**：通常很小（0.01-0.05），因为这只是辅助任务。

### 11.7 总损失函数

完整的损失函数：

$$\mathcal{L}_{total} = \lambda_1 \mathcal{L}_{spec} + \lambda_2 \mathcal{L}_{multi} + \lambda_3 \mathcal{L}_{alpha} + \lambda_4 \mathcal{L}_{LSNR}$$

**设计哲学**：
1. **多目标优化**：不同损失关注不同方面
2. **互补约束**：频谱损失管整体，alpha 损失管细节
3. **多尺度监督**：从粗到细，从整体到局部
4. **感知导向**：损失设计考虑人耳感知特性

这种复杂的损失函数设计，是 DeepFilterNet 能在保持轻量的同时达到高质量的关键原因之一。

## 十二、计算量和性能

### 版本对比

| 版本 | 参数量 | 特点 |
|------|--------|------|
| DeepFilterNet | ~2M | 原版，效果最好 |
| DeepFilterNet2 | ~720K | 轻量版，性能效率平衡好 |
| DeepFilterNet3 | ~420K | 超轻量，适合嵌入式 |

DeepFilterNet2 的 RTF 在普通 CPU 上约 0.15，比 RNNoise（0.07）慢一倍，但比 DCCRN（0.8）快很多。

延迟方面，帧长 10ms，加上处理时间，总延迟大概 10-20ms，满足实时要求。

### DNS Challenge 2022 结果

| 指标 | DeepFilterNet2 |
|------|----------------|
| PESQ | 2.89 |
| SI-SDR | 17.5 dB |
| DNSMOS | 3.82 |

### 消融实验

| 改动 | PESQ 变化 |
|------|----------|
| 去掉 DF 路径 | -0.17 |
| 去掉 ERB 路径 | -0.21 |
| 滤波器长度 5→1（等价于掩码） | -0.17 |
| ERB 频带 32→16 | -0.11 |

说明双路径设计和深度滤波都很重要。滤波器长度 5 是比较好的选择，再长收益很小。

## 十三、与其他方法对比

### vs RNNoise

| | RNNoise | DeepFilterNet |
|---|---------|---------------|
| 定位 | 极致轻量 | 性能效率平衡 |
| 相位 | 不处理 | 隐式处理 |
| 参数量 | 100K | 720K |
| PESQ | 2.54 | 2.89 |
| 适用场景 | 资源极度受限 | 一般实时应用 |

### vs GTCRN

| | GTCRN V1 | DeepFilterNet2 |
|---|---------|----------------|
| 核心思路 | U-Net + 复数掩码 | 双路径 + 深度滤波 |
| 相位处理 | CRM 显式处理 | DF 隐式处理 |
| 参数量 | 139K | 720K |
| DNSMOS | 3.15 | 3.82 |
| 实时支持 | V3 支持 | 支持 |

GTCRN 更轻量，DeepFilterNet 效果更好。

## 十五、总结

DeepFilterNet 的核心贡献：

1. **深度滤波**：用时变 FIR 滤波器代替简单掩码，能隐式处理相位
2. **双路径设计**：ERB 处理全频段粗粒度，DF 处理低频精细
3. **分组 GRU + Shuffle**：减少参数量同时保持信息流通
4. **多分辨率损失**：捕获不同时频尺度的信息

DeepFilterNet 是学习语音增强的好材料：比 RNNoise 复杂，但思路清晰；比 DCCRN 轻量，但效果更好。

对于学习顺序：RNNoise（基础）→ DeepFilterNet（改进）→ GTCRN（另一条路线）。
