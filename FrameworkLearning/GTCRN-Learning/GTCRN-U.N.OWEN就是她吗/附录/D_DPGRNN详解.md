# 附录 D：DPGRNN 详解

DPGRNN = **D**ual-**P**ath **G**rouped **R**ecurrent **N**eural **N**etwork

它是 GTCRN 的瓶颈层，负责把卷积擅长的“局部建模”补齐为“全局时频建模”。

---

## D.1 先回答三个最核心的问题

### 1）DPGRNN 在做什么？

输入特征是 `[B, C, T, F]`（在 GTCRN 里通常是 `[B, 32, T, 55]`）。

DPGRNN 做两件事：

- **沿频率轴建模**（Intra）：每一帧内部，频带之间如何关联
- **沿时间轴建模**（Inter）：每个频带上，历史帧如何影响当前帧

### 2）为什么不只用卷积？

卷积即使用了膨胀率，感受野仍是“有限局部”。
噪声统计变化、语音长程连贯性这类依赖，往往跨越更长时间；谐波关系则跨越整条频率轴。

### 3）为什么不只用一个 RNN？

如果把 `[C, F]` 直接展平给时间 RNN，输入维度变成 `C×F`，参数和计算都很大，而且对频率结构不友好。
双路径把二维问题拆成两个一维问题，更高效也更可控。

---

## D.2 结构总览（一步一步看形状）

```
输入 x: [B, 32, T, 55]

(1) Intra-RNN（频率方向）
    reshape: [B, 32, T, 55] -> [B*T, 55, 32]
    Linear(32->32) -> BiGRU -> Linear(64->32)
    reshape 回来: [B, 32, T, 55]
    残差 + LayerNorm

(2) Inter-RNN（时间方向）
    reshape: [B, 32, T, 55] -> [B*55, T, 32]
    Linear(32->32) -> GRU -> Linear(32->32)
    reshape 回来: [B, 32, T, 55]
    残差 + LayerNorm

输出 y: [B, 32, T, 55]
```

注意：这里 `B*T` 或 `B*55` 只是把“并行样本维”摊平，方便喂给 RNN，不改变建模逻辑。

---

## D.3 Intra 与 Inter 的分工

| 路径 | 序列方向 | 每次处理对象 | 主要学什么 |
|------|----------|--------------|------------|
| Intra | 频率轴 F | 每个时间帧的一条频谱 | 谐波结构、共振峰、跨频带相关性 |
| Inter | 时间轴 T | 每个频带的一段时间序列 | 语音连续性、噪声统计变化、平滑性 |

可以把它记成：

- **Intra：同一时刻，频率之间互相参考**
- **Inter：同一频带，历史帧给当前帧上下文**

---

## D.4 为什么 Intra 用双向，Inter 用单向

### Intra（频率轴）双向 BiGRU

频率轴没有“未来不可见”的约束。低频和高频信息可以彼此帮助，双向更充分。

### Inter（时间轴）单向 GRU

实时降噪需要因果性：当前帧不能看未来帧。单向 GRU 天然满足这个约束。

这也是 GTCRN 能从离线版本平滑过渡到流式版本的关键之一。

---

## D.5 为什么每个 RNN 前后要接 Linear

典型形式：`Linear -> GRU -> Linear`

作用有三点：

1. **输入投影**：把输入映射到更适合 RNN 的子空间
2. **输出回投**：特别是 BiGRU 输出翻倍（2H），需要投回原通道数，方便残差相加
3. **低秩压缩入口**：可把 `32->24` 再进 GRU，大幅减参数

---

## D.6 为什么要残差 + LayerNorm

### 残差

DPGRNN 的目标是“修正特征”，不是从零生成特征。`out = x + f(x)` 通常更容易优化。

### LayerNorm

RNN 输出在不同时间步/频带上分布波动较大。LayerNorm 可稳定训练，减少梯度不稳和层间分布漂移。

---

## D.7 堆叠两个 DPGRNN 的意义

一个 DPGRNN 只完成“一轮”时频交互：先频率、再时间。

堆叠两层后：

- 第 1 层：建立粗粒度时频关联
- 第 2 层：在已融合上下文上做细化判断

这和“多层特征逐步抽象”的思想一致，但比注意力结构更轻量。

---

## D.8 参数量为什么可控（低秩思路）

RNN 参数量大致与隐藏维度平方相关，直接增大 hidden 很贵。

常见压缩法：

```
Linear(32 -> r) -> GRU(r, r) -> Linear(r -> 32), r < 32
```

例如 `r=24`，可显著降参数与算力，同时通常只带来较小性能损失。

这就是“先压缩到低秩子空间建模，再映射回主空间”的思路。

---

## D.9 流式推理时，状态到底怎么管

这是初学者最容易混淆的点：

- **Intra-RNN**：按帧独立处理频率序列，不跨帧，不需要长期状态
- **Inter-RNN**：沿时间推进，需要保存 GRU hidden state

如果网络里有 2 个 DPGRNN，则通常要维护 2 组 Inter hidden states。

部署时这组状态需要：

1. 在音频流开始时初始化（通常全零）
2. 每帧/每 chunk 推理后更新
3. 在断流或说话人切换场景按策略重置

---

## D.10 小白版直觉：DPGRNN 像“两次读表”

把 `[T, F]` 想成一张二维表：

- 第一次按“行”读（每帧一行，沿频率读）
- 第二次按“列”读（每频带一列，沿时间读）

一次行读 + 一次列读，就把二维依赖基本覆盖了。

这就是 Dual-Path 的精髓。

---

## D.11 常见错误与排查

### 错误 1：reshape 维度顺序写反

现象：模型能跑但效果很差。原因是把时间和频率的角色弄反，导致 Intra/Inter 学错对象。

### 错误 2：BiGRU 输出维度没处理

BiGRU 输出通常是 `2H`，若不回投到原维度，残差相加会维度不匹配。

### 错误 3：流式时没保存 Inter hidden state

现象：离线正常，流式出现抖动、断续或明显退化。

### 错误 4：在因果模型里误用双向时间 RNN

这会引入未来信息泄漏，流式场景不可用。

---

## D.12 与 GTConvBlock 的关系（一句话）

- **GTConvBlock**：高效提局部模式 + 动态门控
- **DPGRNN**：补全全局时频依赖

二者不是替代关系，而是前后配合、职责互补。

---

## D.13 一句话总结

DPGRNN 的本质是：

> 把二维时频建模拆成“频率序列建模 + 时间序列建模”两步，在可控参数量下获得全局上下文能力，并保留流式可部署性。
