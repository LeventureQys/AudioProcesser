# 附录 A：U-Net 详解

## A.1 什么是 U-Net

U-Net 最初是 2015 年为医学图像分割设计的网络，因其结构像字母 "U" 而得名。核心思想是**编码器-解码器 + 跳跃连接**。

```
输入
  │
  ▼
编码器（收缩路径）
  │ 逐层下采样，提取高级语义特征
  │ 分辨率降低，通道数增加
  ▼
瓶颈层（最深层，分辨率最低，语义最抽象）
  │
  ▼
解码器（扩展路径）
  │ 逐层上采样，恢复空间分辨率
  │ 分辨率增加，通道数减少
  ▼
输出（与输入同尺寸）
```

---

## A.2 为什么需要跳跃连接

下采样会丢失空间细节：

```
原图 256×256
  ↓ 下采样
128×128     丢失一半像素
  ↓ 下采样
64×64       继续丢失
  ↓ 下采样
32×32       细节几乎没了
  ↓ 上采样
64×64       只能靠"猜"来恢复细节
  ↓ 上采样
256×256     边缘模糊，细节丢失
```

跳跃连接直接把编码器的细节传给解码器，不用从头"猜"：

```
编码器                    解码器
256×256 ─────────────────→ 256×256   细节直接传过来
    ↓                          ↑
128×128 ─────────────────→ 128×128
    ↓                          ↑
64×64 ───────────────────→ 64×64
    ↓                          ↑
32×32 ──────→ 瓶颈 ──────→ 32×32
```

## A.3 U-Net 的完整结构

```
输入                                          输出
  │                                            ↑
  ▼                                            │
┌───┐                                        ┌───┐
│ E1│ ──────────── skip1 ──────────────────→│ D1│
└─┬─┘                                        └─↑─┘
  │ ↓下采样                            上采样↑ │
  ▼                                            │
┌───┐                                        ┌───┐
│ E2│ ──────────── skip2 ──────────────────→│ D2│
└─┬─┘                                        └─↑─┘
  │ ↓下采样                            上采样↑ │
  ▼                                            │
┌───┐                                        ┌───┐
│ E3│ ──────────── skip3 ──────────────────→│ D3│
└─┬─┘                                        └─↑─┘
  │ ↓下采样                            上采样↑ │
  ▼                                            │
┌─────────────────────────────────────────────────┐
│                    瓶颈层                        │
└─────────────────────────────────────────────────┘
```

- 左边：下降的编码器，逐层下采样
- 右边：上升的解码器，逐层上采样
- 横线：跳跃连接
- 底部：瓶颈层，最抽象的特征

---

## A.4 编码器的作用

编码器通过下采样逐步提取高级特征：

```
输入: 高分辨率，低级特征（边缘、纹理）
  ↓ Conv + Pool
第1层: 分辨率/2，开始提取局部模式
  ↓ Conv + Pool
第2层: 分辨率/4，提取更大范围的模式
  ↓ Conv + Pool
第3层: 分辨率/8，提取高级语义（物体、结构）
  ↓
瓶颈: 分辨率最低，语义最抽象
```

类比：浅层看到的是"像素"，深层看到的是"物体"。

---

## A.5 解码器的作用

解码器通过上采样逐步恢复分辨率，每层都用 skip connection 补充细节：

```
瓶颈: 低分辨率，高级语义
  ↓ Deconv + Skip
第1层: 分辨率×2，结合 skip 恢复细节
  ↓ Deconv + Skip
第2层: 分辨率×4，继续恢复
  ↓ Deconv + Skip
第3层: 分辨率×8，接近原始分辨率
  ↓
输出: 与输入同尺寸
```

关键：每层都用 skip connection 补充细节，不是纯靠"猜"。

---

## A.6 跳跃连接的两种融合方式

**拼接（Concatenation）**：原版 U-Net 用这个
- 把 skip 特征和解码器特征在通道维度拼接
- 信息保留完整，网络可以自由选择用哪部分
- 代价：通道数翻倍，需要额外卷积降维

**相加（Addition）**：GTCRN 用这个
- 直接逐元素相加
- 计算量小，不增加通道数
- 要求 skip 和解码器特征维度完全一致

GTCRN 选择相加是因为参数量约束严格，拼接带来的额外参数代价太高。

## A.7 GTCRN 中的 U-Net

GTCRN 把 U-Net 用在语音降噪，输入是 2D 频谱而不是图像：

| 图像 U-Net | GTCRN |
|-----------|-------|
| 2D 图像（H×W） | 2D 频谱（T×F） |
| 空间下采样（高度+宽度） | 只在频率轴下采样 |
| 语义特征（物体、边缘） | 时频特征（语音、噪声模式） |
| 像素级分割 | 频点级掩码 |

**为什么只在频率轴下采样**：时间轴需要保持完整分辨率，因为掩码需要逐帧输出，不能在时间轴上压缩。频率轴可以压缩，因为 ERB 变换已经按感知重要性分配了频率分辨率。

### GTCRN 的 U 形结构

```
频谱 [B, 3, T, 219]
    │
    ▼ DSConv, stride=2
[B, 32, T, 110] ──── skip1 ──────→ + → DSDeconv → [B, 2, T, 219]
    │                                    ↑
    ▼ DSConv, stride=2                   │
[B, 32, T, 55] ───── skip2 ──────→ + → DSDeconv → [B, 32, T, 110]
    │                                    ↑
    ▼ GTConv×6                           │
[B, 32, T, 55] ─── skip3~8 ──────→ + → GTConv×6 → [B, 32, T, 55]
    │                                    ↑
    └──────────→ DPGRNN×2 ───────────────┘
                 （瓶颈层）
```

## A.8 为什么语音降噪适合 U-Net

1. **输入输出同尺寸**：带噪频谱和干净频谱（或掩码）尺寸完全一致，天然适合 U-Net

2. **需要保留细节**：高频细节对音质很重要，下采样会丢失，skip connection 能补回来

3. **需要全局上下文**：降噪需要理解语音内容，区分语音和噪声，深层网络能捕获长程依赖

4. **对称结构自然**：编码器提取时频特征，解码器生成复数掩码，天然对称

## A.9 skip 的逆序使用

解码器逆序使用编码器的 skip：

```
编码顺序: E0 → E1 → E2 → E3 → E4
保存顺序: [0]  [1]  [2]  [3]  [4]

解码顺序: D0 → D1 → D2 → D3 → D4
使用顺序: [4]  [3]  [2]  [1]  [0]

对应关系:
E4（最后编码，最抽象）←→ D0（最先解码，从抽象开始）
E0（最先编码，最具体）←→ D4（最后解码，恢复到具体）
```

这保证了"对应分辨率"的特征相互融合：解码器在恢复到某个分辨率时，使用的是编码器在同一分辨率下保存的特征。
