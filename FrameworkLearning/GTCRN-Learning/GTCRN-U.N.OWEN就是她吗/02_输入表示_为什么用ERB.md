# 02. 输入表示：为什么用 ERB

GTCRN 把 513 个 FFT 频点压缩到 219 个 ERB 频带，这是整个网络的第一步，也是最基础的设计决策。

---

## 一、线性频谱的问题

STFT 输出的是等间距的线性频谱：48kHz 采样、1024 点 FFT，每个频点间隔约 46Hz。这种均匀分配看起来公平，但对语音增强来说是一种浪费。

**人耳对频率的感知不是线性的**：
- 低频（100-500Hz）：人耳分辨率极高，100Hz 和 150Hz 听起来差别很大
- 高频（4000Hz 以上）：人耳分辨率很低，6000Hz 和 6050Hz 几乎听不出差别

线性频谱在高频区域分配了大量频点，但这些频点对人耳来说携带的是冗余信息。用这些冗余频点做计算，是纯粹的浪费。

---

## 二、ERB 的设计哲学

ERB（Equivalent Rectangular Bandwidth）不是一个经验公式，而是基于**人耳听觉滤波器的生理测量**得出的带宽模型：

$$ERB(f) = 24.7 \times (4.37 \times f/1000 + 1)$$

这个公式描述的是：在频率 $f$ 处，人耳的听觉滤波器有多宽。

具体数值：
- 100Hz 处，ERB 约 30Hz（低频，滤波器窄，分辨率高）
- 1000Hz 处，ERB 约 132Hz
- 4000Hz 处，ERB 约 460Hz（高频，滤波器宽，分辨率低）

**ERB 变换的本质**：按照人耳的感知分辨率重新分配频率资源。低频保留精细的频点，高频合并成宽频带。这样每个 ERB 频带对人耳来说"感知价值"是均等的。

---

## 三、为什么是 219 个频带

语音增强任务对频率分辨率的要求比语音识别高得多：
- 语音识别：40-80 个 Mel 频带就够，因为只需要识别音素
- 语音增强：需要重建语音波形，频率细节直接影响音质

219 是一个平衡点：
- 比 513 少很多，后续所有操作的计算量按比例下降（约减少 57%）
- 比 40-80 多很多，保留了足够的频率细节用于高质量重建

**为什么不用更少**：频带太少会导致高频语音细节（辅音、齿音）丢失，降噪后语音听起来"模糊"。

**为什么不用更多**：超过 219 之后，增加的频带大多是人耳感知不到的高频冗余，收益递减。

---

## 四、ERB vs Mel

两者都是感知频率尺度，区别在于：

| | ERB | Mel |
|--|-----|-----|
| 来源 | 听觉滤波器生理测量 | 主观听感实验 |
| 常用频带数 | 100-300 | 40-80 |
| 主要用途 | 语音增强 | 语音识别 |
| 频率分辨率 | 更高 | 较低 |

对语音增强来说，ERB 的优势是可以在保持听觉相关性的同时使用更多频带，重建质量更好。

---

## 五、ERB 变换的工作方式

ERB 变换本质上是一个**固定的线性变换**：用一个预计算好的滤波器矩阵，把 513 维的线性频谱映射到 219 维的 ERB 频带。

```
输入: 线性频谱 [B, T, 513, 2]
         ↓ 矩阵乘法（固定权重）
输出: ERB 频带 [B, T, 219, 2]
```

**为什么权重固定不学习**：ERB 滤波器是基于人耳生理特性的，不需要从数据中学习。固定权重省去了这部分参数，也保证了变换的物理意义不会被训练过程破坏。

**逆变换**：输出时用伪逆矩阵把 219 维恢复到 513 维。伪逆恢复会有轻微的信息损失，但对感知质量影响可以忽略，因为丢失的主要是人耳感知不到的高频冗余。

---

## 六、输入通道的设计

GTCRN 不是直接把 ERB 频谱送入网络，而是拼接三个通道：

```
幅度谱 mag = sqrt(real² + imag²)  → 通道 1
实部   real                        → 通道 2
虚部   imag                        → 通道 3
```

**为什么显式提供幅度谱**：幅度谱直接反映能量分布，是判断"这里有没有语音"最直观的特征。如果只提供实部和虚部，网络需要自己计算 $\sqrt{re^2 + im^2}$，增加了学习难度。显式提供幅度谱相当于给网络一个"捷径"。

**为什么保留实部和虚部**：网络最终要输出复数掩码，需要知道原始的相位状态。只有幅度谱无法恢复相位信息。

**为什么不用幅度谱 + 相位谱**：相位是角度，具有周期性（$-\pi$ 和 $\pi$ 是同一个值），神经网络处理周期性数值很困难。实部+虚部是笛卡尔坐标，没有这个问题。
