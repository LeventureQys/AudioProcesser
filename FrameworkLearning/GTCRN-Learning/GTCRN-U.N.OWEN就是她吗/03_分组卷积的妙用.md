# 03. 深度可分离卷积：为什么要拆开

GTCRN 能在极低参数量下运行，深度可分离卷积是最重要的轻量化手段之一。

---

## 一、标准卷积做了两件事

标准卷积把两件本质上不同的事情混在一起做：

1. **空间特征提取**：在时间轴和频率轴上捕捉局部模式（比如某个频带在相邻帧之间的变化）
2. **通道混合**：把不同通道的信息融合在一起（比如把低频特征和高频特征结合）

参数量是 $C_{in} \times C_{out} \times K^2$，两件事的代价全部叠加在一起。

---

## 二、深度可分离卷积：分开做

深度可分离卷积把这两件事拆成两步：

**第一步：深度卷积（Depthwise Conv）**
- 每个通道独立做空间卷积，不跨通道
- 只负责"在空间上看什么"
- 参数量：$C \times K^2$

**第二步：逐点卷积（Pointwise Conv，即 1×1 卷积）**
- 只做通道混合，不看空间位置
- 只负责"把哪些通道的信息组合在一起"
- 参数量：$C_{in} \times C_{out}$

总参数量：$C \times K^2 + C_{in} \times C_{out}$

对比标准卷积 $C_{in} \times C_{out} \times K^2$，节省比例约为 $1/C_{out} + 1/K^2$。当 $C=32, K=3$ 时，节省约 **8 倍**。

---

## 三、为什么拆开是合理的

空间特征提取和通道混合本来就是两件独立的事：

- **空间特征**：关注的是"这个位置和周围位置的关系"，和通道无关
- **通道混合**：关注的是"不同特征之间的组合"，和空间位置无关

标准卷积把它们混在一起，实际上是在用 $C_{out}$ 倍的参数做同一件空间特征提取的事。深度卷积只做一次，逐点卷积再做通道组合，逻辑更清晰，参数更少。

**信息损失有限**：实验表明，在大多数任务上，深度可分离卷积的效果只比标准卷积略差，但参数量减少了 7-8 倍。这个权衡在轻量化场景下非常值得。

---

## 四、在 GTCRN 中的应用

GTCRN 的 GTConvLite 模块用深度可分离卷积替代了原版的标准卷积：

```
输入 [B, 32, T, F]
    │
    ▼ 深度卷积 (groups=32, k=3×3, dilation=d)
    │ 每个通道独立，捕捉时频局部模式
    │
    ▼ 逐点卷积 (1×1)
    │ 混合通道，融合不同特征
    │
    ▼ BN + PReLU
    │
    ▼ TRALite（时间门控）
    │
    ▼ SEBlock（通道门控）
    │
    + 残差
    │
输出 [B, 32, T, F]
```

**深度卷积的膨胀率**：时间轴用膨胀卷积（dilation=[1,2,4,8,4,2]），频率轴不膨胀（dilation=1）。这是因为需要在时间轴上扩大感受野，而频率轴的感受野已经通过 SFE 和卷积核覆盖。

---

## 五、下采样卷积（DSConv）

编码器的两层下采样也用深度可分离结构：

```
输入 [B, C, T, 219]
    │
    ▼ 深度卷积 (stride=2，频率轴下采样)
    │
    ▼ 逐点卷积
    │
输出 [B, C', T, 110]
```

stride=2 只在频率轴上，时间轴不下采样。原因：
- **频率轴**：需要压缩到 55 个频带，减少后续计算量
- **时间轴**：保持完整的时间分辨率，因为掩码需要逐帧输出

---

## 六、参数量对比

以编码器第一层下采样为例（输入 3 通道，输出 32 通道，k=3×3，频率 219）：

| 卷积类型 | 参数量 | 计算量（MACs/帧） |
|---------|--------|-----------------|
| 标准卷积 | 3×32×9 = 864 | 864×219 ≈ 189K |
| 深度可分离 | 3×9 + 3×32 = 123 | 123×219 ≈ 27K |
| 节省比例 | **7×** | **7×** |

整个网络用深度可分离卷积后，总参数量从原版的约 280K 降到约 139K，计算量同比下降。
